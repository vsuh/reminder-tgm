[![License: BSD-3-Clause](https://img.shields.io/badge/license-BSD--3--Clause-blue.svg)](LICENSE)

Простое приложение для отправки напоминаний в Telegram (и опционально в [ntfy](https://ntfy.sh)) по расписанию на основе `cron`-выражений с дополнительными модификаторами. Приложение состоит из двух процессов:

> После повышения версии проекта, запускается GA workflow обновляющий образ [docker.io/vsuh/cron-tg](https://hub.docker.com/repository/docker/vsuh/cron-tg/general)

Проект можно развернуть в [Docker-контейнере](https://github.com/vsuh/cron-tg-docker.git).

> После повышения версии проекта (тега) автоматически запускается workflow, который обновляет образ [vsuh/cron-tg-docker](https://hub.docker.com/r/vsuh/cron-tg-docker).
>
> ```bash
> git tag v1.1.z
> git push origin master --tags
> ```

## Возможности

- **Гибкое расписание** — стандартные `cron`-выражения плюс дополнительные модификаторы для точной настройки периодичности.
- **Модификаторы** — `d/n` (каждые n дней) и `w/n` (каждые n недель), с возможностью указать начальную дату отсчёта (`YYYYMMDD>`).
- **Несколько чатов Telegram** — каждому расписанию можно назначить свой чат для отправки.
- **Дублирование в ntfy** — расписание можно дополнительно привязать к каналу [ntfy](https://ntfy.sh).
- **Авто-возраст для дней рождения** — если сообщение начинается с `ДР`, а в модификаторе указана дата рождения, бот сам подставит текущий возраст.
- **Shebang-сообщения** — текст уведомления можно подгружать из внешнего JSON-файла по дате (`#!path/to/file.json`).
- **Веб-интерфейс** — добавление, редактирование и удаление расписаний, чатов и каналов ntfy; предпросмотр ближайших срабатываний.
- **Экспорт/импорт расписаний** — выгрузка в JSON и массовая загрузка через API.
- **Резервное копирование БД** — автоматический бэкап SQLite с ротацией и опциональной репликацией по `scp`.
- **Docker** — готовый образ для запуска в контейнере, см. [cron-tg-docker](https://github.com/vsuh/cron-tg-docker.git).

## Архитектура

| Компонент | Файл | Назначение |
|---|---|---|
| Веб-приложение | `web/app.py`, `wsgi.py` | Flask-приложение: UI + REST API |
| Демон рассылки | `rund.py` (точка входа `run.py`) | Проверяет расписания и шлёт сообщения в Telegram/ntfy |
| Логика cron + модификаторов | `lib/cron_utils.py` | Класс `VCron`: валидация и расчёт следующего срабатывания |
| Работа с БД | `lib/db_utils.py` | SQLite: таблицы `schedules`, `chats`, `ntfy_channels` |
| Общие утилиты | `lib/utils.py` | Логирование, загрузка `.env` |
| Шаблоны | `templates/*.html` | HTML-страницы веб-интерфейса |

Веб-приложение и демон рассылки — независимые процессы, использующие общую базу SQLite. Это позволяет обновлять расписания через UI/API, пока демон продолжает работать в фоне.

## Установка и запуск

### Зависимости

Управление зависимостями осуществляется через файлы `requirements/*.txt`:

- `requirements/base.txt` — общие зависимости (`python-dotenv`, `pytz`, `python-dateutil`, `croniter`, `requests`);
- `requirements/web.txt` — дополнительно `Flask`, `gunicorn`, `waitress`;
- `requirements/rund.txt` — только базовые зависимости (для демона).

```bash
pip install -r requirements.txt
```

### Локальный запуск

1. Клонируйте репозиторий:

   ```bash
   git clone https://github.com/vsuh/reminder-tgm.git
   cd reminder-tgm
   ```

2. Создайте виртуальное окружение и установите зависимости:

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   ```

3. Скопируйте файл `env/.env.SAMPLE` в `.env` и настройте переменные окружения (см. [раздел ниже](#переменные-окружения)):

   ```bash
   cp env/.env.SAMPLE .env
   # отредактируйте .env, заменив значения по умолчанию на свои
   ```

4. Запустите веб-приложение:

   ```bash
   ./web_prod.sh
   ```

   либо в режиме отладки (Flask dev-сервер):

   ```bash
   ./web.sh
   ```

5. В отдельном терминале запустите демон рассылки:

   ```bash
   ./rund_prod.sh
   ```

   либо напрямую:

   ```bash
   python rund.py
   # или (legacy-обёртка)
   python run.py
   ```

### Запуск в Docker

Образ собирается отдельным репозиторием [cron-tg-docker](https://github.com/vsuh/cron-tg-docker.git). Внутри контейнера оба процесса (веб и демон) запускаются скриптом `start.sh`, который завершает оба процесса, если один из них упал.

## Переменные окружения

Все переменные читаются из `.env` (или `env/.env.<environment>`, см. `lib/utils.py`). Пример — `env/.env.SAMPLE`.

### Telegram

| Переменная | Назначение |
|---|---|
| `TLCR_TELEGRAM_TOKEN` | Токен бота, выданный `@BotFather` |
| `TLCR_TELEGRAM_CHAT_ID` | ID чата/группы по умолчанию (используется, если в форме не выбран чат) |

### Время

| Переменная | Назначение |
|---|---|
| `TLCR_TZ` | Временная зона сервера, например `Europe/Moscow` |

### База данных

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `TLCR_DB_PATH` | `settings.db` | Путь к файлу SQLite |
| `TLCR_BACKUP_PATH` | `static/db.bak` | Каталог для резервных копий |
| `TLCR_BACKUP_INTERVAL` | `24` | Интервал создания бэкапов, часы |

### scp-репликация бэкапов (опционально, только для `prod`)

| Переменная | Назначение |
|---|---|
| `TLCR_BACKUP_SCP_ODD` / `TLCR_BACKUP_SCP_EVEN` | Цель `scp` (`user@host:/path`) для нечётных/чётных дней года |
| `TLCR_BACKUP_SSH_KEY_PATH` | Путь к приватному ssh-ключу |
| `TLCR_BACKUP_SSH_PORT_ODD` / `TLCR_BACKUP_SSH_PORT_EVEN` | Порт ssh/scp (по умолчанию `22`) |

При ошибке репликации демон отправляет аварийное уведомление в ntfy.

### Логирование

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `TLCR_LOGPATH` | `.` | Каталог для логов |
| `TLCR_LOG_LEVEL` | `INFO` | Уровень логирования: `DEBUG`, `INFO`, `WARNING`, `ERROR` |

### Веб-интерфейс

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `TLCR_SECRET_KEY` | случайное значение | Секретный ключ Flask (обязательно задайте своё значение в production) |
| `TLCR_LIST_ITEMS` | `10` | Сколько ближайших срабатываний показывать на странице `/list/<id>` |
| `TLCR_FLASK_PORT` | `7999` (в Docker — `7878`) | Порт веб-интерфейса |
| `TLCR_FLASK_HOST` | `127.0.0.1` | Адрес, на котором слушает встроенный dev-сервер Flask |
| `TLCR_WEB_USER` / `TLCR_WEB_PASSWORD` | не заданы | Логин/пароль для базовой аутентификации через сессию. Если не задать оба значения — UI и API будут доступны без авторизации |

### Планировщик (демон)

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `TLCR_CHECK_MINUTES` | `60` | Как часто демон проверяет расписания, минуты |

### Режим работы

| Переменная | Назначение |
|---|---|
| `FLASK_ENV` | `development` / `production` |
| `FLASK_DEBUG` | `0` / `1` |

> Имя окружения (`dev`/`prod`) для загрузки `env/.env.<name>` определяется автоматически по ОС (`lib/utils.get_environment_name`): `prod` для Linux/macOS, `dev` — для прочих. Если в корне проекта есть файл `.env`, он имеет приоритет.

Рекомендации для production:

- использовать абсолютные пути для файлов и директорий;
- отключить режим отладки (`FLASK_DEBUG=0`, `FLASK_ENV=production`);
- задать собственный `TLCR_SECRET_KEY`;
- задать `TLCR_WEB_USER`/`TLCR_WEB_PASSWORD`, если интерфейс смотрит в интернет;
- настроить подходящий уровень логирования.

## Использование

### Веб-интерфейс

Доступен по адресу `http://host:<TLCR_FLASK_PORT>` (по умолчанию `7999`, в Docker-образе — `7878`).

Основные страницы:

| URL | Назначение |
|---|---|
| `/` | Список расписаний: создание, сортировка, переход к редактированию/удалению |
| `/edit/<id>` | Редактирование расписания |
| `/list/<id>` | Предпросмотр ближайших `N` срабатываний расписания |
| `/message/<id>` | Просмотр содержимого JSON-файла для shebang-сообщений |
| `/chats` | Управление чатами Telegram (добавление/удаление) |
| `/ntfy` | Управление каналами ntfy (добавление/изменение/удаление) |
| `/export` | Выгрузка всех расписаний в JSON-файл |
| `/login`, `/logout` | Авторизация (если задан `TLCR_WEB_USER`/`TLCR_WEB_PASSWORD`) |

> **Важно:** перед добавлением расписания нужно сначала создать хотя бы один чат на странице `/chats` — без этого добавление расписания завершится ошибкой.

## Модификаторы расписания

Модификаторы уточняют срабатывание `cron`-выражения (минутная часть `cron` при этом игнорируется — проверка идёт с точностью до часа/дня).

| Модификатор | Значение |
|---|---|
| `d/n` | Срабатывает каждые `n` дней |
| `w/n` | Срабатывает каждые `n` недель (отсчёт по дням недели от базовой даты) |
| `YYYYMMDD>d/n` или `YYYYMMDD>w/n` | То же самое, но отсчёт периода ведётся от указанной даты, а не от условной точки отсчёта |

Примеры:

- `d/3` — каждые 3 дня от внутренней базовой даты;
- `20240301>d/3` — каждые 3 дня, начиная с 1 марта 2024 года;
- `20250526>d/4` — каждые 4 дня, начиная с 26 мая 2025 года (см. `static/dataschedules.json`).

Без модификатора (пустая строка) срабатывание определяется только `cron`-выражением.

## Shebang-сообщения

Если текст сообщения начинается с `#!`, оставшаяся часть строки трактуется как путь к JSON-файлу:

```
#!relative/path/to/file.json
```

В момент срабатывания приложение:

1. читает указанный JSON-файл;
2. ищет в нём запись, дата которой совпадает с текущей датой;
3. использует значение поля `text` этой записи как итоговый текст уведомления.

Если подходящей записи на сегодня не найдено — отправляется исходная строка как есть (в логах фиксируется предупреждение).

Формат JSON-файла:

```json
[
  {
    "date": "2025-12-19",
    "text": "С Днём снабженца!"
  },
  {
    "date": "2025-12-26",
    "text": "С днём ангела-хранителя!"
  }
]
```

- `date` — дата в формате `YYYY-MM-DD`;
- `text` — текст сообщения для этой даты.

Просмотреть содержимое подключённого файла можно через `/message/<id>` в веб-интерфейсе.

## Дни рождения

Если текст сообщения начинается с `ДР`, а в поле `modifier` указана дата рождения в формате `YYYYMMDD` (с необязательным периодом после `>`, например `19900101>d/365`), приложение автоматически вычисляет возраст на момент срабатывания и добавляет его в скобках:

```
ДР Иван Иванов (36 лет)
```

## Уведомления через ntfy

Любому расписанию можно назначить канал [ntfy](https://ntfy.sh) (поле `ntfy_id`). При срабатывании уведомление отправляется и в Telegram, и в указанный ntfy-топик. Управление каналами — на странице `/ntfy`:

- `name` — произвольное название канала;
- `url` — полный URL топика ntfy (например, `https://ntfy.sh/my-topic`);
- `title` — заголовок уведомления (опционально; если содержит не-ASCII символы, заголовок `Title` не отправляется — это ограничение HTTP-заголовков).

## API

Приложение предоставляет REST API для управления расписаниями. Полное описание — в [API.md](API.md). Пример массовой загрузки расписаний из файла:

```bash
curl -X POST -H "Content-Type: application/json;charset=utf-8" \
  --data-binary @dataschedules.json \
  http://host:7878/schedules_all
```

Пример файла — [static/dataschedules.json](static/dataschedules.json).

## Резервное копирование БД

Демон (`rund.py`) автоматически создаёт резервную копию SQLite-базы каждые `TLCR_BACKUP_INTERVAL` часов в каталоге `TLCR_BACKUP_PATH`, оставляя только 3 последних файла. В `prod`-окружении, если настроены `TLCR_BACKUP_SCP_ODD`/`TLCR_BACKUP_SCP_EVEN`, копия дополнительно реплицируется по `scp` на один из двух хостов (выбор зависит от чётности дня года). При ошибке репликации отправляется уведомление в служебный ntfy-топик.

## Локальные git hooks

Все хуки проекта хранятся в каталоге `.githooks`. При клонировании репозитория для разработки выполните один раз:

- Linux/macOS:

  ```bash
  .githooks/init-hooks
  ```

- Windows:

  ```powershell
  powershell -ExecutionPolicy Bypass -File .githooks\init-hooks.ps1
  ```

После этого Git будет использовать хуки из папки `.githooks/`.

### Тестирование цепочки workflow обновления образа

Для ручного запуска процесса обновления Docker-образа:

```bash
curl -X POST https://api.github.com/repos/vsuh/cron-tg-docker/dispatches \
  -H "Accept: application/vnd.github.v3+json" \
  -H "Authorization: token ${GH_TOKEN}" \
  -d '{"event_type":"new-reminder-tgm-tag","client_payload":{"tag":"v99.0.0-dispatch-test","message":"Test from curl"}}'
```

Токен должен иметь права `repo.*`. В результате в Telegram должно прийти сообщение вида:

> ✅ Build successful!
> docker.io/vsuh/cron-tg:v99.0.0-dispatch-test
> Update notify-docker.yml - Test from curl
